\# 週末コードレビュー特訓 記録



週末を利用して取り組んだ、コードレビューとリファクタリングの特訓記録用リポジトリです。

「動くコード」から「安全で保守しやすいプロのコード」へ引き上げるためのベストプラクティスを蓄積しています。



\## 📂 ディレクトリ構成ルール



新しい課題に取り組むたびに、連番でディレクトリを追加していきます。（例: `01\_xxx`, `02\_yyy`）

各課題のディレクトリ内は、以下のルールで統一されています。



\* `README.md` : その課題の出題日、NGポイント、改善策の解説

\* `bad/` : レビュー前のNGコード（セキュリティホールやパフォーマンス課題が潜むコード）

\* `good/`: レビュー・修正後の正解コード（アーキテクチャや設計を改善したコード）



※ 課題ごとの詳細な学びは、各ディレクトリ内の `README.md` を参照してください。
※ create_task.batは簡易的なディレクトリを作るバッチです



\## 🤖 出題の進め方



今後のお題生成・レビュー評価・模範解答は Claude Code（テックリード役）が担当します。

役割・進行フロー（出題／レビュー評価／模範解答の3フェーズ）・記録ルールは `CLAUDE.md` に定義しています。

「お題を出して」と言えば新しい課題が始まります。

# 第8問：支払い金額を計算するサービス（テストのしやすさ）

**📅 出題日**: 2026年7月12日

## 📋 レビュー課題

ECサイトで、商品の「支払い金額」を計算する `PriceService` です。
ベース価格に対して、時間帯・タイムセール・会員ランクに応じた割増や割引を順番に適用し、
最終的な支払い金額を返します。

- 深夜(22時〜翌5時)なら 10% 割増
- タイムセール中なら 20% オフ
- GOLD 会員なら 100 円引き

ローカルで動かすと期待通りの金額が返ってきて、一見きちんと動いています。

対象: `bad/PriceService.java`

このコードを本番に出す前提で、「どこが問題か・どう直すべきか」をレビューしてみてください。
今回は特に **「このロジックをどうやって自動テストで守るか（テスト容易性）」** の観点で見てみましょう。

> ヒント：「深夜割増が正しく効くか」「セール中の金額が正しいか」「GOLD会員だけ100円引かれるか」を
> あなたが JUnit で確かめようとしたら、素直にテストが書けるでしょうか？書きにくいなら、それは**設計のにおい**です。

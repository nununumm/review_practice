# 第11問：外部決済ゲートウェイ呼び出しサービス（ログと例外の扱い）

**📅 出題日**: 2026年7月19日

## 📋 レビュー課題

外部の決済サービス（カード決済のゲートウェイ）に、カード決済を依頼する `PaymentGatewayService` です。
`charge()` を呼ぶと決済を実行し、成功なら `true` / 失敗なら `false` を返します。
あとから決済トラブルを調査できるように、開始時・レスポンス時・失敗時にログを残しています。

ローカルでテスト用カードを使うと決済が通り、失敗時にも `false` が返り、
ログにもしっかり記録が残っていて、一見きちんと動いています。

対象: `bad/PaymentGatewayService.java`

このコードを本番に出す前提で、「どこが問題か・どう直すべきか」をレビューしてみてください。
今回は特に **「ログに何をどう残すか」と「例外（失敗）をどう扱うか」** の観点で見てみましょう。

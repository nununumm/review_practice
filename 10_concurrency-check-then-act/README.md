# 第10問：抽選キャンペーンの応募受付サービス（同時アクセス）

**📅 出題日**: 2026年7月19日

## 📋 レビュー課題

抽選キャンペーンの応募を受け付ける Spring Boot のサービスです。
`apply(userId)` を呼ぶと、そのユーザーの応募を記録します。
**1ユーザーにつき1回だけ**応募できるように、応募済みかどうかをチェックしています。

ローカルで1人ずつ順番に叩くと、2回目はちゃんと「すでに応募済みです」と弾かれ、
合計応募数もきれいに増えていって、一見きちんと動いています。

対象: `bad/CampaignApplicationService.java`

このサービスは人気キャンペーンの一部で、**開始と同時に大量の応募が同時に押し寄せる**場面で使われます。
「どこが問題か・どう直すべきか」をレビューしてみてください。
今回は特に **「同時アクセスに耐えられるか（並行処理・スレッド安全性）」** の観点で見てみましょう。

# 第12問：パスワード再設定の申請サービス（テストのしやすさ）

**📅 出題日**: 2026年7月19日

## 📋 レビュー課題

ユーザーが「パスワードを忘れた」ときに、再設定を申請する `PasswordResetService` です。
`requestReset(email)` を呼ぶと、①再設定用のトークン（合言葉）を発行し、
②有効期限（1時間）を付けて保存し、③そのリンクをメールで送る、という処理をします。

ローカルで動かすと、トークン付きのメールがちゃんと飛んできて、一見きちんと動いています。

対象: `bad/PasswordResetService.java`

このコードを本番に出す前提で、「どこが問題か・どう直すべきか」をレビューしてみてください。
今回は特に **「このサービスをどうやって自動テストで守るか（テスト容易性）」** の観点で見てみましょう。

> ヒント：「発行されたトークンが正しく保存されるか」「有効期限は本当に1時間後か」「ちゃんとメールを送ろうとするか」を
> あなたが JUnit で確かめようとしたら、素直にテストが書けるでしょうか？書きにくいなら、それは**設計のにおい**です。
